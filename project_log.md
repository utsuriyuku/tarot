# 塔罗牌AI解读工具 - 项目开发日志

## 项目愿景
制作一个具备优秀交互体验和视觉效果（如Three.js粒子特效）的塔罗牌抽取与解读工具，并接入大模型（LLM）进行动态的解牌分析。

## 开发阶段记录
### 第一轮讨论 (2026-05-12)
- **初始需求确定**：
  - 核心功能：塔罗牌抽取、大模型解牌分析。
  - 视觉交互：强调前端表现，计划引入 Three.js 等方案实现粒子特效与沉浸感。
  - 项目管理：在工作区维护Markdown进度日志，并保持多轮对话总结。
- **当前进度**：
  - 建立项目日志文档。
  - 等待与用户确认前端技术栈与整体架构框架。

### 第二轮讨论 (2026-05-12)
- **系统设定升华**：
  - 确定项目的哲学内核：“时间本身不存在”，“未来决定现在”（类似量子物理中的逆因果律或波函数坍缩）。
  - 已将该设定具象化为 `system_prompt.md` 并将其暴露在项目根目录，方便用户随时调优。
- **基础设施建设**：
  - 创建了完善的项目说明 `README.md`。
  - 准备采用 React + Vite + React-Three-Fiber 作为前端脚手架建立基础工程。

### 第三轮讨论 (2026-05-12)
- **UI交互与体验优化**：
  - 明确“页面是美丽的表皮，交互是用户的眼睛”的设计导向。前端组件必须兼顾“美感”和“可理解性”。
  - 解答了开发层面的“配置化（JSON）”疑问：向用户阐明 JSON 是开发代码组织数据的形式（维护78张卡牌），不需要用户自己写代码。
  - **实现 API 挂载节点（配置弹窗）**：引入毛玻璃质感的前端遮罩面板，内敛且优雅的输入框（API URL、API Key、Model），状态存储到 `localStorage` 中。文案包装为“观测节点连接”，融入世界观场景。

### 第四轮讨论 (2026-05-12)
- **核心塔罗业务流闭环**：
  - 创建了 `tarotData.ts` 数据集，定义了核心卡面和量子视角的对应释义（目前已包含愚者、魔术师、命运之轮等大阿卡纳牌用作测试）。
  - 创建了 `llm.ts` 请求服务，集成我们设定好的 System Prompt，组装用户抽牌数据的 payload 发往 OpenAI / 兼容的大模型 API。
  - 完善了 `TarotCard.tsx`：加上了逆位判定，当卡牌被判定为逆位时，翻转动画会在Z轴上额外旋转180度，以视觉化展示“逆位”。
  - 引入了 `InterpretationPanel.tsx`（解读面板）：点击中间的卡牌时，标题会逐渐隐退，右侧优雅滑出一个采用赛博/克苏鲁神秘感排版的解牌框，呈现“打字机 (Typewriter)”的文字喷吐表现效果。如果并未配好 API，它会优雅地引导用于点击“终端配置”。

### 第五轮讨论 (2026-05-12)
- **卡面贴图加载支持**：处理了待办[A. 真实图片材质]，并利用 Vite / ThreeJS TextureLoader 技术配合 React Suspense 实现。
- **意念交互前置**：处理了待办[B. 抽牌前输入]。编写了底部输入框。目前业务链路调整为：必须要用户先凝神打出/写出“执念或疑团”，确认锁定意念后，画面会从幽暗之中微微变亮，3D卡牌才被解锁，允许抽选。这个交互将用户心念作为真正连接 AI 的桥梁。
- **扩充 78 张牌库**：完成了待办[C. 扩充完整78张牌JSON数据]。编写了一个 Node.js 脚本一键生成了包含 22 张大阿卡纳（带有维基真实图照源）和 56 张小阿卡纳的大型数据表 `tarotData.ts`。项目彻底成型。
- **全球公网发布**：使用 GitHub Actions 配置了全自动流水线（CI/CD），通过云渲染直接向 GitHub Pages 投放静态结果。至此整个项目架构完备，支持后续敏捷更迭。
